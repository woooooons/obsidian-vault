# OS: 우분투 24.04 desktop 버전 
## 1단계: 시스템 업데이트

웹 서버 설치 전, 패키지 인덱스를 최신 상태로 유지.

의존성 충돌과 보안 이슈를 피하기 위한 기본 절차.

```bash
sudo apt update
sudo apt upgrade -y

```

---

## 2단계: Apache 설치

Ubuntu 24.04의 기본 저장소에는 `apache2` 패키지가 포함되어 있다.

```bash
sudo apt install apache2 -y

```

설치 시 자동으로 서비스가 등록되고 기본 설정이 배치.

---

## 3단계: Apache 서비스 상태 확인

Apache는 systemd 서비스로 동작.

```bash
sudo systemctl status apache2

```

정상 상태라면 `active (running)` 이 표시된다.

필요 시 직접 제어:

```bash
sudo systemctl start apache2
sudo systemctl stop apache2
sudo systemctl restart apache2
sudo systemctl enable apache2   # 서버 부팅 시 자동 실행

```

---

## 4단계: 방화벽(UFW) 설정

Ubuntu Server에서는 UFW가 기본 방화벽이다.

웹 서버는 **80(http), 443(https)** 포트 개방이 필수다.

```bash
sudo ufw allow 'Apache Full'
sudo ufw reload

```

확인:

```bash
sudo ufw status

```

---

## 5단계: 웹 서버 접속 테스트

서버의 IP 주소 확인:

```bash
ip a

```

브라우저에서 접속:

```
<http://서버IP>

```

정상이라면 **Apache2 Ubuntu Default Page**가 표시된다.

이 시점에서 이미 웹 서버는 동작 중이다.

---

## 6단계: 웹 문서 루트(DocumentRoot) 구조 이해

기본 웹 루트:

```
/var/www/html

```

기본 페이지:

```
/var/www/html/index.html
```

## 7단계: 실제 돌리는 서버

```
# 기본 html 파일 삭제
sudo rm -rf /var/www/html/*

# 실제 돌리는 서버 폴더 및 파일을 아파치 경로에 복사 
sudo cp -r demo/* /var/www/html/

# 권한 주기
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html

```
