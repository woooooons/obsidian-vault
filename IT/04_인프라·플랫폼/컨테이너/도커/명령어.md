## 1. 이미지(Image) 관리 명령

### 검색 / 다운로드 / 리스트 / 삭제 / 정보 조회

| 명령                                | 설명                      |
| --------------------------------- | ----------------------- |
| `docker search <이름>`              | Docker Hub에서 이미지 검색     |
| `docker pull <이미지>:<태그>`          | 원격 저장소에서 이미지 다운로드       |
| `docker images`                   | 로컬 이미지 목록               |
| `docker rmi <이미지>`                | 이미지 삭제                  |
| `docker rmi -f <이미지>`             | 강제 삭제                   |
| `docker image prune`              | 사용하지 않는 이미지 삭제          |
| `docker image prune -a`           | dangling + unused 전체 정리 |
| `docker build -t <이름:태그> .`       | Dockerfile 기반 이미지 빌드    |
| `docker build --no-cache -t <이름>` | 캐시 없이 새로 빌드             |
| `docker tag <이름:태그> <새이름:새태그>`    | 이미지 태그 변경               |
| `docker history <이미지>`            | 이미지 레이어 구조 확인           |
| `docker inspect <이미지>`            | 이미지 상세 정보 출력            |

## 2. 컨테이너(Container) 관리 명령

### 생성 / 실행

|명령|설명|
|---|---|
|`docker run <이미지>`|생성 + 실행|
|`docker run -d <이미지>`|백그라운드 실행|
|`docker run --name <이름> <이미지>`|이름 지정|
|`docker run -it <이미지> bash`|컨테이너 터미널 접속 실행|
|`docker run -p 8080:80 <이미지>`|포트 매핑|
|`docker run -e KEY=VALUE <이미지>`|환경 변수 전달|
|`docker run -v /host:/container <이미지>`|볼륨 마운트|
|`docker run --restart=always <이미지>`|자동 재시작 정책|

### 상태 조회

| 명령                      | 설명         |
| ----------------------- | ---------- |
| `docker ps`             | 실행 중인 컨테이너 |
| `docker ps -a`          | 전체 컨테이너    |
| `docker logs <컨테이너>`    | 로그 조회      |
| `docker logs -f <컨테이너>` | 실시간 로그     |

### 접속 / 명령 실행

|명령|설명|
|---|---|
|`docker exec -it <컨테이너> bash`|bash 접속|
|`docker exec <컨테이너> <명령>`|명령 실행|
|`docker attach <컨테이너>`|메인 프로세스에 붙기|

### 정지 / 시작 / 재시작

|명령|설명|
|---|---|
|`docker stop <컨테이너>`|정지|
|`docker start <컨테이너>`|시작|
|`docker restart <컨테이너>`|재시작|
|`docker kill <컨테이너>`|강제 종료 (SIGKILL)|

### 삭제 / 정리

| 명령                       | 설명             |
| ------------------------ | -------------- |
| `docker rm <컨테이너>`       | 삭제             |
| `docker rm -f <컨테이너>`    | 실행 중이라도 삭제     |
| `docker container prune` | 중지된 컨테이너 전체 삭제 |

## 3. 네트워크(Network) 명령

|명령|설명|
|---|---|
|`docker network ls`|네트워크 리스트|
|`docker network inspect <네트워크>`|상세 정보|
|`docker network create <이름>`|사용자 정의 네트워크 생성|
|`docker network rm <이름>`|네트워크 삭제|
|`docker network prune`|사용되지 않는 네트워크 삭제|
|`docker network connect <네트워크> <컨테이너>`|네트워크에 컨테이너 추가|
|`docker network disconnect <네트워크> <컨테이너>`|네트워크에서 제거|

## 4. 볼륨(Volume) 명령

|명령|설명|
|---|---|
|`docker volume ls`|볼륨 리스트|
|`docker volume create <이름>`|볼륨 생성|
|`docker volume inspect <이름>`|볼륨 상세 정보|
|`docker volume rm <이름>`|볼륨 삭제|
|`docker volume prune`|사용 안 하는 볼륨 전체 삭제|

## 5. 파일 복사(Copy)

|명령|설명|
|---|---|
|`docker cp <컨테이너>:<경로> <호스트>`|컨테이너 → 호스트 파일 복사|
|`docker cp <호스트> <컨테이너>:<경로>`|호스트 → 컨테이너 파일 복사|

## 6. 시스템 모니터링 / 최적화

|명령|설명|
|---|---|
|`docker stats`|실시간 CPU/메모리/IO 상황|
|`docker top <컨테이너>`|컨테이너 프로세스 목록|
|`docker inspect <컨테이너>`|모든 설정·정보 조회|
|`docker events`|실시간 이벤트 스트림|
|`docker system df`|디스크 사용량 분석|
|`docker system prune`|전체 불필요 자원 삭제|
|`docker system prune -a`|이미지·컨테이너·네트워크 모두 삭제|

## 7. Registry 관련 명령 (실전에서 필수)

|명령|설명|
|---|---|
|`docker login`|Docker Hub 또는 Registry 로그인|
|`docker logout`|로그아웃|
|`docker push <이미지>`|원격 저장소로 이미지 업로드|
|`docker pull <이미지>`|원격에서 다운로드|
|`docker tag <기존> <새태그>`|태그 변경 후 push|

## 8. Docker Compose 관련 기본 명령 (실전 필수)

| 명령                       | 설명                   |
| ------------------------ | -------------------- |
| `docker compose up`      | 전체 서비스 실행            |
| `docker compose up -d`   | 백그라운드 실행             |
| `docker compose down`    | 모든 컨테이너 종료 및 네트워크 삭제 |
| `docker compose ps`      | Compose 관리 컨테이너 조회   |
| `docker compose logs`    | 로그 조회                |
| `docker compose logs -f` | 실시간 로그               |
| `docker compose restart` | 전체 재시작               |
