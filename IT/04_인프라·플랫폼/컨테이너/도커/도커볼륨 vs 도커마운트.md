## 1. Docker Volume

### 1.1 정의

Volume은 Docker 엔진이 직접 관리하는 데이터 저장 영역이다.

- 실제 물리 위치는 Linux 기준 `/var/lib/docker/volumes/` 아래에 생성된다.
    
- Docker가 생성·삭제·백업·마이그레이션 등을 관리한다.
    

### 1.2 특징

1. **Docker 전용 관리 영역**  
    운영체제의 일반 파일 구조와 분리되어 있으며 Docker CLI로 제어 가능하다.
    
2. **권한 및 보안 분리**  
    호스트 시스템 파일을 직접 수정하는 방식이 아니므로 의도치 않은 파일 손상 위험이 작다.
    
3. **보다 높은 이식성**  
    디렉터리 구조가 독립적이라 서버 이동 또는 배포가 비교적 용이하다.
    
4. **Plugin 지원 가능**  
    NFS, CIFS, 클라우드 스토리지 드라이버 등 외부 스토리지와 연동 가능하다.
    

### 1.3 사용 예

- DB 데이터(MySQL, PostgreSQL 등)
    
- Docker Swarm 또는 Kubernetes 환경
    
- 멀티 컨테이너가 공유해야 하는 데이터

## 2. Bind Mount

### 2.1 정의

Bind mount는 호스트 OS의 특정 디렉터리 또는 파일을 컨테이너 내부 경로로 직접 연결하는 방식이다.

- 개발자가 지정한 경로가 그대로 사용된다.
    

### 2.2 특징

1. **호스트 파일에 직접 접근**  
    컨테이너가 호스트 파일을 읽고, 수정하고, 삭제할 수 있다.  
    이는 유연하지만 리스크가 존재한다.
    
2. **높은 실시간성**  
    코드 변경 → 컨테이너에 즉시 반영되기 때문에 개발 환경에서 많이 사용된다.
    
3. **Docker 관리 대상 아님**  
    Docker CLI로 관리할 수 없고 호스트 OS에서 직접 관리해야 한다.
    
4. **호스트 디렉터리 의존**  
    절대 경로 기반이므로 이동성·이식성이 떨어진다.
    

### 2.3 사용 예

- 로컬 소스코드를 컨테이너에 실시간 적용
    
- 설정 파일을 컨테이너에서 관리하지 않고 호스트에서 유지하려는 경우
    
- 디버깅 및 로깅 파일 호스트 저장

## 3. 비교 표

|구분|Docker Volume|Bind Mount|
|---|---|---|
|관리 주체|Docker 엔진|호스트 OS|
|위치|`/var/lib/docker/volumes`|개발자가 지정한 경로|
|이식성|높음|낮음|
|보안|격리 수준 높음|호스트 파일 직접 접근으로 위험도 존재|
|성능|Docker에서 최적화됨|호스트 디스크 성능에 직접 의존|
|백업·복구|Docker 명령으로 가능|OS 기반으로 직접 수행|
|활용 환경|운영환경, 데이터베이스|개발환경, 실시간 소스 반영|

---

## 4. 어떤 상황에서 무엇을 써야 하는가

### 4.1 Volume이 더 적합한 경우

1. 운영환경에서 안정성이 필요할 때
    
2. 중요한 데이터를 다루는 DB 컨테이너
    
3. 여러 컨테이너가 동일 데이터를 공유할 때
    
4. 마이그레이션과 백업이 필요한 경우
    

### 4.2 Bind Mount가 적합한 경우

1. 개발 환경에서 코드 변경을 즉시 테스트해야 하는 경우
    
2. IDE에서 작성 중인 파일을 컨테이너가 바로 읽어야 할 때
    
3. 설정 파일을 외부에서 계속 변경해 적용해야 할 때
    

---

## 5. 보안 및 운영 관점 분석

### 5.1 Volume

- 컨테이너와 디렉터리가 격리되어 있어 보안상 안전한 구조
    
- Docker가 권한을 관리하므로 의도치 않은 파일 변경 위험이 낮음
    

### 5.2 Bind Mount

- 읽기/쓰기 권한이 호스트 파일에 그대로 반영
    
- 잘못된 경로 지정 시 호스트 시스템 파일이 손상될 가능성
    
- 운영환경에서는 거의 사용하지 않는 이유
    

---

## 6. 추천 전략

- **개발 환경:** Bind Mount
    
- **운영 환경:** Volume
    
- **DB·로그 등 영구적 데이터:** Volume
    
- **일시적 테스트 파일:** Bind Mount
