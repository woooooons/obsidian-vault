## 1. 초기 설정

### Git 설치 확인

```bash
git --version
```

### 사용자 정보 설정

```bash
git config --global user.name "이름"
git config --global user.email "이메일@example.com"
```

### 설정 확인

```bash
git config --list
git config user.name
```

## 2. 저장소 생성 및 복제

### 새 저장소 초기화

```bash
git init
```

### 원격 저장소 복제

```bash
git clone <저장소-URL>
git clone <저장소-URL> <디렉토리명>
```

## 3. 기본 작업 흐름

### 파일 상태 확인

```bash
git status
git status -s  # 간단한 형식
```

### 파일 추가 (Staging)

```bash
git add <파일명>
git add .  # 모든 변경사항
git add *.js  # 특정 패턴
git add -p  # 대화형 추가
```

### 커밋

```bash
git commit -m "커밋 메시지"
git commit -am "메시지"  # add + commit
git commit --amend  # 마지막 커밋 수정
```

### 변경사항 확인

```bash
git diff  # Working directory vs Staging
git diff --staged  # Staging vs 마지막 커밋
git diff HEAD  # Working directory vs 마지막 커밋
git diff <브랜치1> <브랜치2>
```

## 4. 브랜치 관리

### 브랜치 조회

```bash
git branch  # 로컬 브랜치 목록
git branch -r  # 원격 브랜치 목록
git branch -a  # 모든 브랜치 목록
```

### 브랜치 생성 및 전환

```bash
git branch <브랜치명>  # 브랜치 생성
git checkout <브랜치명>  # 브랜치 전환
git checkout -b <브랜치명>  # 생성 + 전환
git switch <브랜치명>  # 브랜치 전환 (최신)
git switch -c <브랜치명>  # 생성 + 전환 (최신)
```

### 브랜치 병합

```bash
git merge <브랜치명>
git merge --no-ff <브랜치명>  # Fast-forward 방지
git merge --squash <브랜치명>  # 커밋 압축
```

### 브랜치 삭제

```bash
git branch -d <브랜치명>  # 안전 삭제
git branch -D <브랜치명>  # 강제 삭제
git push origin --delete <브랜치명>  # 원격 브랜치 삭제
```

## 5. 원격 저장소 작업

### 원격 저장소 관리

```bash
git remote  # 원격 저장소 목록
git remote -v  # 상세 정보
git remote add <이름> <URL>  # 원격 저장소 추가
git remote remove <이름>  # 원격 저장소 제거
git remote rename <기존이름> <새이름>
```

### Push (업로드)

```bash
git push origin <브랜치명>
git push -u origin <브랜치명>  # upstream 설정
git push --all  # 모든 브랜치
git push --tags  # 모든 태그
git push --force  # 강제 푸시 (주의)
```

### Fetch & Pull (다운로드)

```bash
git fetch  # 원격 변경사항 가져오기
git fetch origin
git pull  # fetch + merge
git pull --rebase  # fetch + rebase
```

## 6. 히스토리 관리

### 로그 확인

```bash
git log
git log --oneline  # 한 줄로 표시
git log --graph  # 그래프 형태
git log --all --decorate --oneline --graph
git log -n 5  # 최근 5개
git log --author="이름"
git log --since="2024-01-01"
git log --grep="키워드"
git log <파일명>  # 특정 파일 히스토리
```

### 커밋 되돌리기

```bash
git reset --soft HEAD~1  # 커밋 취소, 변경사항 유지
git reset --mixed HEAD~1  # 커밋 + add 취소
git reset --hard HEAD~1  # 완전 취소 (주의)
git reset <커밋해시>
```

### 특정 커밋 제거

```bash
git revert <커밋해시>  # 새 커밋으로 되돌림
git revert HEAD  # 마지막 커밋 되돌림
```

### Rebase

```bash
git rebase <브랜치명>
git rebase -i HEAD~3  # 대화형 rebase
git rebase --continue  # 충돌 해결 후 계속
git rebase --abort  # rebase 취소
```

## 7. Stash (임시 저장)

```bash
git stash  # 변경사항 임시 저장
git stash save "메시지"
git stash list  # stash 목록
git stash apply  # 최근 stash 적용
git stash apply stash@{n}  # 특정 stash 적용
git stash pop  # 적용 + 삭제
git stash drop stash@{n}  # 특정 stash 삭제
git stash clear  # 모든 stash 삭제
```

## 8. 태그 관리

```bash
git tag  # 태그 목록
git tag <태그명>  # Lightweight 태그
git tag -a <태그명> -m "메시지"  # Annotated 태그
git tag -a <태그명> <커밋해시>  # 특정 커밋에 태그
git show <태그명>  # 태그 정보 확인
git push origin <태그명>  # 태그 푸시
git push origin --tags  # 모든 태그 푸시
git tag -d <태그명>  # 로컬 태그 삭제
git push origin :refs/tags/<태그명>  # 원격 태그 삭제
```

## 9. 파일 관리

### 파일 삭제

```bash
git rm <파일명>  # 파일 삭제 + staging
git rm --cached <파일명>  # Git에서만 제거
```

### 파일 이동/이름 변경

```bash
git mv <기존파일명> <새파일명>
```

### 파일 무시

```bash
# .gitignore 파일 생성
echo "node_modules/" >> .gitignore
git add .gitignore
```

## 10. 고급 기능

### Cherry-pick (특정 커밋 가져오기)

```bash
git cherry-pick <커밋해시>
git cherry-pick <커밋1>..<커밋2>
```

### Blame (코드 수정 이력)

```bash
git blame <파일명>
git blame -L 10,20 <파일명>  # 10-20번 줄
```

### Bisect (버그 찾기)

```bash
git bisect start
git bisect bad  # 현재 커밋이 나쁨
git bisect good <커밋해시>  # 이 커밋은 좋음
git bisect reset  # 종료
```

### Submodule

```bash
git submodule add <저장소-URL> <경로>
git submodule update --init --recursive
git submodule foreach git pull origin main
```

### Worktree

```bash
git worktree add <경로> <브랜치명>
git worktree list
git worktree remove <경로>
```

## 11. 정보 확인

```bash
git show <커밋해시>  # 커밋 상세 정보
git show HEAD  # 마지막 커밋
git reflog  # 모든 참조 로그
git ls-files  # 추적 중인 파일 목록
git ls-tree --name-only HEAD  # HEAD의 파일 목록
```

## 12. 정리 및 최적화

```bash
git clean -n  # 삭제될 파일 미리보기
git clean -f  # 추적되지 않는 파일 삭제
git clean -fd  # 디렉토리까지 삭제
git gc  # 가비지 컬렉션
git prune  # 불필요한 객체 제거
```

## 13. 유용한 별칭 (Alias)

```bash
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.lg "log --graph --oneline --all"
```

## 14. 충돌 해결

```bash
# 충돌 발생 시
git status  # 충돌 파일 확인
# 파일 수동 편집 후
git add <파일명>
git commit

# 병합 취소
git merge --abort

# 충돌 해결 도구
git mergetool
```

## 참고사항

- `HEAD`: 현재 브랜치의 최신 커밋
- `HEAD~1`: 이전 커밋
- `HEAD^`: 첫 번째 부모 커밋
- `origin`: 기본 원격 저장소 이름
- `main`/`master`: 기본 브랜치 이름

커밋 메시지 작성 권장사항:

- 첫 줄은 50자 이내로 요약
- 본문은 72자마다 줄바꿈
- 현재형 동사 사용 (Add, Fix, Update 등)
