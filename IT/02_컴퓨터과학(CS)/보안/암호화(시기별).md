## 1. 고전 암호 시대 (기원전 ~ 1940년대)

### 카이사르 암호 (기원전 100년경)

**개념**

- 알파벳을 일정 칸수만큼 이동
- 예: A→D, B→E (3칸 이동)
- "HELLO" → "KHOOR"

**특징**

```
• 치환 암호의 원조
• 가능한 키: 25개 (알파벳 개수-1)
• 수작업으로 몇 분 내 해독 가능
```

**역사적 의의**

- 최초의 체계적 암호화 방법
- 군사 통신에 사용됨

---

### 비제네르 암호 (1553년)

**개념**

- 키워드를 반복하여 각 문자마다 다른 이동
- 키: "KEY" → K(10), E(4), Y(24) 이동 반복
- 카이사르 암호보다 훨씬 강력

**특징**

```
• 다중 알파벳 치환 암호
• 400년간 "해독 불가능"으로 여겨짐
• 1863년 카시스키에 의해 해독법 발견
```

---

### 에니그마 (1918-1945)

**개념**

- 독일군이 제2차 세계대전에서 사용
- 회전 로터 기계식 암호화 장치
- 매일 설정 변경 (로터 순서, 초기 위치)

**특징**

```
• 가능한 설정: 약 10^23가지
• 앨런 튜링과 폴란드 암호학자들이 해독
• 컴퓨터 과학 발전의 계기
```

**붕괴 이유**

- 독일군의 운영상 실수 (반복 패턴)
- 연합군의 암호 해독 기계 (Bombe) 개발
- 수학적 약점 발견

## 2. 초기 컴퓨터 시대 (1950-1970년대)

### DES (Data Encryption Standard, 1977)

**탄생 배경**

- IBM이 개발한 Lucifer 알고리즘 기반
- 미국 NIST가 1977년 공식 표준 채택
- 최초의 공개 표준 암호화 알고리즘

**기술 사양**

```
• 대칭키 블록 암호
• 블록 크기: 64비트
• 키 길이: 56비트 (패리티 8비트 포함 총 64비트)
• 라운드: 16회
• Feistel 구조
```

**사용 기간**

```
1977-1998: 공식 표준
1998-2005: 보안 취약 경고
2005년: 공식 폐기
```

**보안 문제**

```
• 56비트 키: 너무 짧음 (2^56 = 7200조)
• 1998년: EFF의 Deep Crack이 56시간 만에 해독
• 1999년: 22시간 만에 해독
• 무차별 대입 공격에 취약
```

**현재 상태**

```
X 절대 사용 금지
X 레거시 시스템에 일부 남아있음
```

---

### 3DES (Triple DES, 1998)

**탄생 이유**

- DES의 56비트 키가 너무 약함
- 새 알고리즘 개발 시간 필요
- 기존 DES 하드웨어 재활용

**동작 방식**

```
EEE 방식 (3개 키 모두 다름):
  암호문 = E_K3(D_K2(E_K1(평문)))
  
EDE2 방식 (K1 = K3):
  암호문 = E_K1(D_K2(E_K1(평문)))
```

**기술 사양**

```
• 키 길이: 168비트 (3개 키 × 56비트)
• 실효 보안 강도: 112비트 (중간자 공격 고려)
• 처리 속도: DES의 1/3
```

**사용 기간**

```
1998-2023: 금융권 주력
2023년: NIST 공식 폐기 권고
2030년: 완전 사용 금지 예정
```

**문제점**

```
• 느린 속도 (3번 암호화)
• 64비트 블록 → Sweet32 공격 취약
• 레거시 호환 목적으로만 사용
```

**현재 상태**

```
점진적 폐기 중
신규 시스템 사용 금지
AES로 마이그레이션 필요
```

---

### RC4 (1987)

**개념**

- Ron Rivest가 개발한 스트림 암호
- 간단하고 빠른 구현

**기술 사양**

```
• 스트림 암호 (블록 암호 아님)
• 키 길이: 40-2048비트 (가변)
• 바이트 단위 암호화
```

**전성기**

```
• WEP (Wi-Fi 보안)
• SSL/TLS (초기 버전)
• Microsoft Office 암호화
```

**보안 문제 발견**

```
2001: 초기 키스트림 편향 발견
2013: RC4 실용적 공격 가능 입증
2015: IETF가 TLS에서 RC4 금지
```

**현재 상태**

```
X 완전히 폐기됨
X 어떤 용도로도 사용 금지
```

## 3. 현대 암호화 태동기 (1980-1990년대)

### RSA (1977)

**혁명적 개념**

- 최초의 실용적 공개키 암호 시스템
- Ron Rivest, Adi Shamir, Leonard Adleman 개발
- 소인수분해 난이도 기반

**동작 원리**

```
키 생성:
  1. 두 큰 소수 p, q 선택
  2. n = p × q 계산
  3. 공개키: (e, n)
  4. 개인키: (d, n)
  
암호화: C = M^e mod n
복호화: M = C^d mod n
```

**키 길이 변천**

```
1977: 512비트 (당시 충분)
1990년대: 1024비트 권장
2010년대: 2048비트 표준
현재: 2048-4096비트
```

**현재 상태**

```
여전히 광범위하게 사용
양자 컴퓨터 위협
• HTTPS 인증서
• SSH 키
• 디지털 서명
```

---

### MD5 (1991)

**개념**

- Message Digest 5
- Ron Rivest 개발
- 128비트 해시 출력

**전성기 용도**

```
• 파일 무결성 검증
• 비밀번호 해싱
• 체크섬
```

**보안 붕괴**

```
1996: 충돌 가능성 이론적 발견
2004: 실제 충돌 생성 성공
2008: 가짜 SSL 인증서 생성 (Flame 멀웨어)
```

**충돌 예시**

```
두 개의 서로 다른 파일이 동일한 MD5 해시를 가질 수 있음
→ 보안 문제 (악성 파일을 정상 파일로 위장 가능)
```

**현재 상태**

```
X 암호학적 용도 금지
X 보안 목적 사용 불가
비보안 체크섬으로만 사용 (파일 다운로드 확인 등)
```

---

### SHA-1 (1995)

**개념**

- Secure Hash Algorithm 1
- NSA 설계, NIST 표준
- 160비트 해시 출력

**사용 기간**

```
1995-2010: 광범위 사용
2010-2017: 점진적 폐기
2017: Google이 실제 충돌 생성 (SHAttered)
2020: 완전 폐기
```

**붕괴 과정**

```
2005: 이론적 공격 가능성
2017: SHAttered - 실제 충돌 PDF 생성
  • 계산량: 6500 CPU 년 + 110 GPU 년
  • 비용: 약 11만 달러
2020: 선택된 접두사 충돌 공격 (더 실용적)
```

**영향받은 시스템**

```
• Git (2017년 이전)
• SVN
• SSL/TLS 인증서
• 디지털 서명
```

**현재 상태**

```
X 완전히 폐기됨
X 모든 브라우저에서 차단
SHA-256 이상으로 마이그레이션 완료
```

## 4. 현대 암호화 시대 (2000년대 ~ 현재)

### AES (Advanced Encryption Standard, 2001)

**탄생 배경**

```
1997: NIST가 DES 후속 알고리즘 공모
1998-2000: 15개 후보 중 5개 최종 후보
  • MARS (IBM)
  • RC6 (RSA)
  • Rijndael (벨기에)
  • Serpent (영국/이스라엘)
  • Twofish (미국)
  
2000: Rijndael 선정
2001: AES로 공식 표준화
```

**선정 이유**

```
• 뛰어난 보안성
• 최고 수준의 성능
• 간결한 구조
• 하드웨어/소프트웨어 구현 용이
```

**기술 사양**

```
블록 크기: 128비트 (고정)
키 길이: 128, 192, 256비트

AES-128: 10 라운드
AES-192: 12 라운드
AES-256: 14 라운드

구조: 치환-순열 네트워크 (SPN)
```

**보안 상태**

```
현재까지 실용적 공격 없음
양자 컴퓨터 내성 (일부)
 • AES-128: 양자 시대 64비트 보안
 • AES-256: 양자 시대 128비트 보안
2050년까지 안전 예측
```

**현재 활용**

```
• 모든 HTTPS 통신
• Wi-Fi (WPA2, WPA3)
• VPN
• 디스크 암호화
• 파일 암호화
• 데이터베이스 암호화
```

---

### SHA-2 패밀리 (2001)

**종류**

```
SHA-224: 224비트 (드물게 사용)
SHA-256: 256비트 가장 널리 사용
SHA-384: 384비트
SHA-512: 512비트 고보안
SHA-512/224: SHA-512의 224비트 버전
SHA-512/256: SHA-512의 256비트 버전
```

**SHA-256 상세**

```
• 출력: 256비트 (32바이트)
• 블록 크기: 512비트
• 라운드: 64회
• 보안 강도: 2^128 (생일 공격 고려)
```

**사용 사례**

```
• 비트코인 채굴
• SSL/TLS 인증서
• Git 커밋 해시
• 파일 무결성 검증
• 디지털 서명
• 블록체인
```

**현재 상태**

```
완전히 안전
업계 표준
공격 발견 안 됨
2040년까지 안전 예측
```

---

### SHA-3 (Keccak, 2015)

**탄생 배경**

```
2007: NIST가 SHA-2 백업용 알고리즘 공모
  (SHA-1 붕괴 대비)
2012: Keccak 선정
2015: SHA-3로 공식 표준화
```

**SHA-2와의 차이**

```
SHA-2: Merkle-Damgård 구조
SHA-3: Sponge 구조 (스폰지 함수)

SHA-3은 SHA-2의 대체가 아닌 백업용
```

**종류**

```
SHA3-224, SHA3-256, SHA3-384, SHA3-512
SHAKE128, SHAKE256 (가변 길이 출력)
```

**현재 상태**

```
안전함
아직 SHA-2만큼 널리 사용 안 됨
• 일부 블록체인 프로젝트
• 미래 대비용
```

---

### ChaCha20 (2008)

**개념**

- Daniel J. Bernstein 개발
- Salsa20의 개선 버전
- 스트림 암호

**특징**

```
- 키 길이: 256비트
- Nonce: 96비트
- 소프트웨어 최적화 (AES-NI 없어도 빠름)
- ARM, 모바일에서 특히 우수
```

**ChaCha20-Poly1305**

```
- ChaCha20: 암호화
- Poly1305: 인증 (MAC)
- AEAD (인증 암호화)
- TLS 1.3 지원
```

**현재 활용**

```
- Google Chrome (TLS)
- Android 전체 디스크 암호화
- WireGuard VPN
- Signal 메신저
- SSH
```

**vs AES-GCM**

```
ChaCha20-Poly1305:
  • 소프트웨어 구현 빠름
  • 모바일/IoT 우수
  • 타이밍 공격 내성

AES-GCM:
  • 하드웨어 가속 시 빠름
  • Intel/AMD CPU 최적
  • 표준화 먼저 됨
```

**현재 상태**

```
현대 표준
AES의 대안
모바일 환경 1순위
```

---

### Curve25519 / Ed25519 (2006)

**개념**

- Daniel J. Bernstein 설계
- 타원곡선 암호 (ECC)
- RSA보다 짧은 키로 동일 보안

**종류**

```
Curve25519 (X25519):
  • 키 교환용 (ECDH)
  • 128비트 보안 강도
  • 256비트 키

Ed25519:
  • 디지털 서명용 (EdDSA)
  • 128비트 보안 강도
  • 빠른 서명/검증
```

**보안 강도 비교**

```
RSA-2048   = 112비트 보안 = Curve25519
RSA-3072   = 128비트 보안
RSA-15360  = 256비트 보안
```

**장점**

```
- 매우 빠름 (RSA 대비 10-100배)
- 짧은 키 (32바이트 vs RSA 256바이트)
- 타이밍 공격 내성
- 구현 단순
```

**현재 활용**

```
- SSH (ssh-ed25519)
- TLS 1.3
- Signal, WhatsApp
- Tor
- WireGuard
- GPG
```

**현재 상태**

```
RSA 대체 가속화
현대 시스템 표준
양자 컴퓨터에 취약 (RSA와 동일)
```

## 5. 암호화 모드의 진화

### ECB (Electronic Codebook) - 1970년대

```
상태: X 사용 금지
문제: 패턴 노출
```

### CBC (Cipher Block Chaining) - 1976

```
상태: 레거시 호환용
문제: 패딩 오라클, 순차 처리
사용: 파일 암호화, 레거시 TLS
```

### CTR (Counter) - 1979

```
상태: 현역
장점: 병렬 처리, 랜덤 액세스
사용: VPN, 디스크 암호화
```

### GCM (Galois/Counter Mode) - 2005

```
상태: 현대 표준
장점: 인증 + 암호화, 병렬 처리
사용: TLS 1.3, HTTPS, WPA3
```

## 6. 비밀번호 해싱의 진화

### MD5 (1991)

```
X 완전히 깨짐
X 절대 사용 금지
```

### SHA-1 (1995)

```
X 깨짐
X 사용 금지
```

### SHA-256 (2001)

```
비밀번호 해싱용으론 부적합
- 너무 빠름 → 무차별 대입 쉬움

```

### bcrypt (1999)

```
현재 표준 중 하나

- 기본 내장 Salt
```

### scrypt (2009)

```
메모리 집약적
- bcrypt보다 강력
- ASIC 공격 어려움
```

### Argon2 (2015)

```
최신 표준
- Password Hashing Competition 우승
- 메모리 + CPU 집약적
- Argon2id 권장
```

**현재 권장 사항**

```
신규 시스템: Argon2id
레거시: bcrypt (cost 12 이상)
절대 금지: MD5, SHA-1, 단순 SHA-256
```

## 8. 현재 (2025년) 권장 사항

### 대칭키 암호화

```
1순위: AES-256-GCM
2순위: ChaCha20-Poly1305
3순위: AES-128-GCM

절대 금지: DES, 3DES, RC4, ECB 모드
```

### 해시 함수

```
범용: SHA-256, SHA-512
비밀번호: Argon2id, bcrypt

절대 금지: MD5, SHA-1
```

## 요약

핵심

- MD5, SHA-1, DES는 한때 표준이었지만 지금은 완전히 깨짐
- AES, SHA-256은 20년 넘게 안전하지만 영원하지 않음
- 암호화 알고리즘의 수명은 보통 20-30년
- 정기적인 보안 감사와 업그레이드가 필수

할것

1. 레거시 시스템의 암호화 방식을 즉시 확인
2. MD5, SHA-1, DES, 3DES 발견 시 최우선 교체
3. 신규 시스템은 AES-256-GCM + SHA-256을 사용
4. 양자 컴퓨터 시대를 대비 (5-10년 내)
