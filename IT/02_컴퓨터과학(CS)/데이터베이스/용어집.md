## 1. 데이터 모델링 / 설계

### 데이터 모델(Data Model)

현실 세계의 데이터를 구조화하여 표현한 개념적 틀

### 개념 모델(Conceptual Model)

업무 개념 중심의 최상위 모델
엔티티와 관계만 정의

### 논리 모델(Logical Data Model)

업무 규칙 기반 데이터 구조
컬럼, PK/FK는 있으나 DBMS 종속 요소 없음.

### 물리 모델(Physical Data Model)

특정 DBMS 기준의 실제 구현 모델
데이터 타입, 인덱스, 제약조건 포함

### ERD(Entity Relationship Diagram)

엔티티 간 관계를 시각화한 도식.

### 정규화(Normalization)

중복 제거와 무결성 강화를 위한 설계 기법

### 반정규화(Denormalization)

성능 향상을 위해 의도적으로 중복을 허용하는 설계

## 2. 엔티티 / 테이블 구조

### 엔티티(Entity)

업무적으로 관리해야 할 데이터 대상

### 테이블(Table)

엔티티의 물리적 구현체

### 컬럼(Column)

테이블의 속성 단위

### 로우(Row)

테이블의 단일 데이터 레코드

### 도메인(Domain)

컬럼이 가질 수 있는 값의 범위

---

## 3. 키(Key) 체계

### 기본키(Primary Key)

레코드를 유일하게 식별하는 키

### 외래키(Foreign Key)

다른 테이블의 PK를 참조하는 키

### 후보키(Candidate Key)

PK 후보가 될 수 있는 키

### 대체키(Alternate Key)

선택되지 않은 후보키

### 복합키(Composite Key)

둘 이상의 컬럼으로 구성된 키

### 인공키(Surrogate Key)

업무 의미 없는 시스템 생성 키

### 자연키(Natural Key)

업무적으로 의미 있는 키

---

## 4. 데이터 타입

### 문자형

CHAR, VARCHAR, TEXT 등 문자열 저장 타입

### 숫자형

INT, BIGINT, DECIMAL, FLOAT 등

### 날짜형

DATE, TIME, TIMESTAMP, DATETIME 등

### 이진형

BLOB, BYTEA 등 바이너리 데이터

### Boolean

TRUE/FALSE 값 저장 타입

## 5. 제약조건(Constraint)

### NOT NULL

NULL 값 허용 안 함.

### UNIQUE

중복 값 허용 안 함.

### CHECK

조건식에 맞는 값만 허용

### DEFAULT

기본값 자동 입력

### 참조 무결성

FK를 통한 관계 무결성 보장

---

## 6. 인덱스(Index)

### 인덱스

검색 성능 향상용 구조

### Unique Index

중복 허용 안 함.

### Composite Index

다중 컬럼 인덱스

### Clustered Index

데이터 저장 순서를 결정

### Non-Clustered Index

데이터와 분리된 인덱스

### Covering Index

쿼리를 인덱스만으로 처리 가능

## 10. 동시성 / 격리

### 격리 수준

동시 처리 간 간섭 허용 범위

- READ UNCOMMITTED
    
- READ COMMITTED
    
- REPEATABLE READ
    
- SERIALIZABLE
    

### Dirty Read

커밋 안 된 데이터 조회

### Non-Repeatable Read

같은 쿼리 결과 달라짐.

### Phantom Read

조회 범위 내 행 변화 발생.
